# 指标体系 V1.0

V1.0 先建立指标分组、数据结构和少量基础工程指标，不实现完整探测模型、匹配滤波、模糊函数或抗干扰仿真。

## 指标分组

- 探测性能：用于描述波形在指定场景中的目标探测能力。
- 抗干扰性能：用于描述波形面对压制、欺骗、扫频等干扰时的性能保持能力。
- 反侦察性能：用于描述波形被截获、识别和参数估计的风险水平。
- 工程可实现性：用于描述带宽、时宽、采样率、功率、复杂度等工程约束。

## 已实现基础工程指标

### 平均功率

- 指标含义：描述复 IQ 信号在采样窗口内的平均功率水平。
- 计算公式：`average_power = mean(abs(iq)^2)`。
- 单位：W。
- 指标方向：在相同约束下通常不是单独越大越好，需要结合探测距离、发射功率限制和反侦察需求解释。
- 第一版简化假设：直接基于离散复基带采样计算，不建模天线增益、传播损耗和硬件效率。

### 峰值功率

- 指标含义：描述复 IQ 信号在采样窗口内出现的最大瞬时功率。
- 计算公式：`peak_power = max(abs(iq)^2)`。
- 单位：W。
- 指标方向：用于检查发射机峰值能力约束，过高可能增加硬件实现压力。
- 第一版简化假设：直接使用采样点最大值，不建模 DAC、功放饱和和带外失真。

### PAPR

- 指标含义：峰均功率比，用于描述信号峰值功率相对平均功率的抬升程度。
- 计算公式：`PAPR_dB = 10 * log10(peak_power / average_power)`。
- 单位：dB。
- 指标方向：通常越低越有利于功放效率和工程实现。
- 第一版简化假设：只基于离散采样计算，不进行过采样峰值补偿。

### TBP

- 指标含义：时间带宽积，用于描述波形时宽和带宽共同决定的处理潜力。
- 计算公式：`TBP = bandwidth_hz * pulse_width_s`。
- 单位：无量纲。
- 指标方向：在脉冲压缩等处理场景中，通常更大的 TBP 表示更高的潜在处理增益，但也会增加工程实现压力。
- 第一版简化假设：只使用配置中的名义带宽和脉宽，不验证实际频谱占用。

### 平均功率谱密度 PSD

- 指标含义：描述平均功率在名义带宽内的平均分布强度。
- 计算公式：`PSD = average_power_w / bandwidth_hz`。
- 单位：W/Hz。
- 指标方向：在平均功率不变时，PSD 越低通常越有利于低截获特性，但需要结合探测性能解释。
- 第一版简化假设：假设功率在名义带宽内均匀分布，不计算真实频谱形状。

## 结果表达

每个指标使用 `MetricValue` 表达，包含指标名称、数值、单位和说明。每个评估维度使用 `AxisScore` 表达，包含维度名称、得分和指标列表。完整评估使用 `EvaluationResult` 表达。

## 约束

- 指标计算必须在 `radar_eval_core` 中实现。
- UI 层只展示和组织结构化结果。
- 大语言模型只能解释结构化指标结果，不允许直接计算雷达指标。
